# ⚡️ Vibe Monitor

**The Ultimate Analytics Layer for Claude Coding.**

Vibe Monitor is a real-time performance tracking and visualization suite designed specifically for power users of Claude (Desktop & CLI). It turns the "black box" of AI coding into a transparent, data-driven experience, helping you understand exactly how Claude thinks, uses tools, and generates code.

![Vibe Monitor Dashboard](https://github.com/robin-ph/vibe-monitor/raw/main/screenshot_placeholder.png) *(Add your real screenshot here!)*

## 🚀 Features

- **Real-time Metrics**: Track Tokens per Second (TPS), thinking duration, and tool execution time as they happen.
- **Native macOS App**: A sleek, lightweight Swift/SwiftUI menu bar application for at-a-glance monitoring.
- **Deep Claude Integration**: Specifically optimized to parse and visualize Claude's internal reasoning and multi-step tool calls.
- **Historical Analysis**: Archive and review your coding sessions to identify performance patterns or bottlenecks.
- **Visual Breakdown**: Dynamic stacking bar charts splitting time into:
  - 🟣 **Thinking**: Claude's reasoning phase.
  - 🟡 **Tool Use**: File system, terminal, and browser interactions.
  - 🔵 **Output**: Final response generation.
  - 🔴 **Network/Overhead**: Latency and processing delays.

## 🏗️ Architecture

- **Backend (Python/FastAPI)**: The heart of the system. It monitors logs, calculates metrics, and provides a WebSocket/REST API for clients.
- **Web Dashboard (HTML5/Canvas)**: A feature-rich visualization center for deep dives into turn-by-turn data.
- **Native App (Swift/SwiftUI)**: A premium macOS experience using AppKit + SwiftUI for high-performance, low-overhead monitoring.

## 🏁 Getting Started

### 1. Backend & Web Dashboard
```bash
# Install dependencies
pip install fastapi uvicorn pydantic

# Start the server
python server.py
```
Open `http://localhost:8000` to see the web dashboard.

### 2. macOS Native App
1. Open the `VibeMonitorSwift` folder in **Xcode**.
2. Build and run the project.
3. The ⚡️ icon will appear in your menu bar.

## 🛠️ Configuration

The monitor works by listening to the log activity generated by Claude. Ensure your environment is set up to output logs that the `hook_monitor.py` or `server.py` can capture.
